# ♻️ Return To Me - 재활용 분류 수거기 PRD

## 1. 제품 개요 (Product Overview)
**Return To Me**는 딥러닝 기반의 객체 분류 기술을 활용하여 사용자가 분리수거를 올바르게 할 수 있도록 돕는 실시간 재활용 판별 솔루션입니다. 웹캠 및 모바일 카메라를 통해 재활용품을 비추면 해당 물품의 종류와 재활용 가능 여부(오염도 기반)를 즉각적으로 피드백합니다.

### 1-1. 배경 및 취지
- 환경 보호 및 자원 재활용의 중요성 증대
- 복잡한 분리수거 기준으로 인한 사용자 혼란 해소
- 올바른 분리수거 문화를 정착시키고 재활용 샵 이용 포인트 등 혜택 제공 독려

## 2. 제품 목표 (Goals)
1.  **실시간성**: 웹캠/카메라 입력을 실시간으로 처리하여 지연 없는 결과 제공.
2.  **정확성**: 플라스틱, 캔, 종이, 유리 등 주요 재활용품에 대한 높은 분류 성능 확보.
3.  **사용 편의성**: 복잡한 조작 없이 카메라 노출만으로 직관적인 시각 피드백 제공.
4.  **확장성**: PoC 단계의 웹캠 데모를 시작으로 최종적인 모바일 앱 이식 구조 확보.

## 3. 타겟 사용자 (Target Audience)
- 올바른 분리수거를 실천하고자 하는 일반 시민
- 환경 교육이 필요한 어린이 및 학생
- 재활용 정거장 및 무인 수거 기기를 운영하는 지자체/업체

## 4. 핵심 기능 (Key Features)
### F1: 2단계 실시간 추론 파이프라인 (Detection & Classification)
- 1단계 (Detection): YOLOv8-Nano 모델을 활용해 화면 내 쓰레기 객체의 위치를 실시간으로 탐지하고 바운딩 박스(Bounding Box) 추출
- 2단계 (Classification): 추출된 영역(Cropped Tensor)을 MobileNetV3 Small 모델로 전달하여 재질(플라스틱, 캔, 종이 등) 및 오염도에 따른 재활용 가능 여부를 최종 판별

### F2: 재활용 가능 여부 판별 (오염도 체크)
- 객체의 상태(깨끗함 vs 오염됨)를 이진 분류(Binary Classification)로 판별.
- "재활용 가능" 또는 "재활용 불가능" 결과를 사용자에게 제공.

### F3: 시각적 피드백 제공 (UI/UX)
- 비디오 스트림 위에 추론 결과(클래스명, 재활용 여부, 신뢰도 Score)를 오버레이(Overlay) 표시.
- 결과에 따른 색상 구분 (예: 가능 - 녹색, 불가능 - 적색).

### F4: 신뢰도(Confidence) 표시
- 모델의 예측 결과와 함께 신뢰도 점수를 표시하여 오분류 가능성 안내.

### F5: 재활용 샵 연동 (Recycle Shop Integration)
- 사용자가 올바르게 분리수거한 물품을 재활용 샵에 가져가면 포인트 적립.
- 앱 내에서 적립된 포인트 확인 및 사용 가능.

## 5. 기술 아키텍처 (Technical Architecture)
### 5-1. 시스템 흐름도 (Sequence Diagram)
제품의 데이터 처리 흐름은 아래와 같습니다 (제시된 시퀀스 다이어그램 기반):
1.  **User**: 촬영 시작 요청.
2.  **Webcam UI Control**: 프레임 캡처 루프 시작.
3.  **Frame Capturer**: RGBA 포맷의 프레임 캡처 후 프로세서 전달.
4.  **Frame Processor**: 추론용 전처리(FP16/INT8 등) 및 영역 크롭.
5.  **Inference Engine**: 분류 모델(Classification Model)에 추론 요청.
6.  **Classification Model**: 종류 및 재활용 여부 결과 반환.
7.  **Result Displayer**: 최종 결과와 시각화 메타데이터를 원본 화면에 오버레이하여 사용자에게 출력.

### 5-2. 기술 스택
- **언어**: Python
- **딥러닝**: PyTorch, torchvision, timm
- **영상 처리**: OpenCV
- **UI 프레임워크**: Streamlit / Gradio (PoC용)
- **최적화**: ONNX / TFLite (향후 모바일 이식용)
- **Frontend**: Stitch 이용하여 UI 디자인 후, Antigravity에 반영

## 6. 개발 로드맵 (Roadmap)
- **1단계 (PoC)**: 노트북 웹캠 기반 유리/종이/플라스틱/캔 분류 데모 완성. (현재 단계)
- **2단계 (고도화)**: 오염도 판별 알고리즘 고도화.
- **3단계 (이식)**: 모바일 앱(Android/iOS) 환경 최적화 및 배포.
